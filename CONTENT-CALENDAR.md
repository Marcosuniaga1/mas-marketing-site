## Flujo semanal de publicación

Cada lunes, seguir este proceso:

1. **Generar el artículo** — Usar las skills `/content-strategy` + `/humanizer` con el título, enfoque y servicio relacionado de la semana. Revisar tono de voz, datos y CTA antes de publicar.
2. **Generar la imagen** — Copiar el prompt de imagen de la semana y pegarlo en Gemini o ChatGPT. Descargar en 1200x630, guardar como `/public/blog/[slug-del-articulo].jpg`.
3. **Crear el archivo MDX** — Guardarlo en `content/blog/[slug].mdx` con el frontmatter completo:
   ```yaml
   ---
   title: "Título del artículo"
   slug: "slug-del-articulo"
   date: "2026-XX-XX"
   excerpt: "Extracto de 1-2 líneas para SEO y cards"
   category: "Categoría del badge"
   featuredImage: "/blog/slug-del-articulo.jpg"
   relatedService: "/servicio-correspondiente"
   ---
   ```
4. **Verificar build** — Ejecutar `npm run build` y confirmar que compila sin errores.
5. **Commit y push** — `git add . && git commit -m "feat(blog): add [título corto]" && git push`
6. **Compartir en redes** — Publicar en Instagram y WhatsApp Status con link al artículo.
